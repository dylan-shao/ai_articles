## Quantifying infrastructure noise in agentic coding evals
- Source: Anthropic Engineering
- Author: N/A
- Publish date: N/A
- URL: https://www.anthropic.com/engineering/infrastructure-noise

Anthropic 展示了在 SWE-bench、Terminal-Bench 一类 agentic coding 评测中，CPU、RAM、时间预算、容器调度方式与资源 enforcement 方法本身，会显著改变最终得分。在其 Terminal-Bench 2.0 内部实验里，最佳与最差基础设施配置之间可出现 6 个百分点的差距，甚至超过模型榜单之间常见的胜负差。

Why it matters: 这不是一篇泛泛而谈的“评测不完美”文章，而是把 infra 变成一等变量来分析。对任何正在比较模型、构建内部 coding eval、或把 benchmark 分数用于采购与 rollout 决策的团队而言，文章的核心信息是：如果 runtime 条件没有被严格标准化并可复现，那么你测到的可能是 harness 与平台工程能力，而不是模型能力。

Key takeaways:
- Agentic coding eval 与静态基准不同，运行环境是问题求解过程的一部分，因此资源预算差异会改变“考试本身”。
- 仅给出推荐资源规格并不足够；资源限制如何被 enforced、容器如何被调度、超时如何设置，都会改变 benchmark 实际测量的对象。
- 对于内部评测体系，必须把 infra config、调度策略、失败分类与 error budget 纳入 eval 元数据，否则跨团队、跨时间的结果不可比较。

## GitHub Copilot CLI now supports Copilot auto model selection
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-17T00:00:00+00:00
- URL: https://github.blog/changelog/2026-04-17-github-copilot-cli-now-supports-copilot-auto-model-selection/

GitHub 在 Copilot CLI 中正式提供自动模型选择，系统会根据套餐、策略与限流情况，在 GPT-5.4、GPT-5.3-Codex、Sonnet 4.6、Haiku 4.5 等模型之间动态路由，并在 CLI 中显示最终使用的模型。

Why it matters: 这是一个值得关注但技术含量有限的产品变化：它反映 CLI 级 agent tooling 正在从“用户手选模型”转向“按策略自动路由模型”。对企业平台团队而言，重点不在功能本身，而在可观测性、成本归因、以及策略控制是否足够细。

Key takeaways:
- CLI 代理正在引入与聊天产品相同的自动路由层，模型选择越来越像平台策略而非用户界面设置。
- 当前披露的信息主要集中在可用性与计费折扣，缺少关于路由逻辑、任务类型适配和效果评测的技术细节。

## Themes

- AI coding 评测正在从模型比较转向“模型 + runtime + harness”的系统比较。
- 平台与基础设施细节已成为 benchmark 可复现性的核心变量。
- 开发者工具开始默认采用自动模型路由，但技术透明度仍不足。

## Implications

- 如果你的组织用外部榜单做模型采购或上线决策，应要求 benchmark 提供完整 runtime 配置、失败分类、资源限制和复现实验脚本。
- 内部 AI coding eval 平台应把 infra 版本、容器规格、调度机制、超时策略与重试策略写入实验记录，并区分模型失败与基础设施失败。
- 对于 CLI/IDE agent 产品，接下来需要重点审查自动路由带来的成本波动、可解释性和审计能力，而不是只看是否“支持更多模型”。
