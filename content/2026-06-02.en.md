## Updates to GitHub Copilot billing and plans
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-01
- URL: https://github.blog/changelog/2026-06-01-updates-to-github-copilot-billing-and-plans/

GitHub announced that as of June 1, all Copilot plans now use AI Credits-based usage billing. Copilot code review consumes both AI Credits and GitHub Actions minutes, with new user-level budget controls and upgrades to Copilot Max also introduced.

Why it matters: This is more than pricing news; it signals a transition from seat-based AI tooling to metered infrastructure consumption. For platform teams, budget controls, runner planning, chargeback, and usage governance become core parts of AI coding adoption.

Key takeaways:
- Copilot adoption will increasingly resemble cloud resource governance, requiring budgets, quotas, default runners, and usage policy.
- Teams enabling Copilot code review should plan for Actions capacity alongside direct AI spend.

Tags: GitHub Copilot, AI billing, platform governance, developer tooling
Scores: signal 7/5, novelty 5/5, actionability 8/5

## Evaluation models in auto for individual plans
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-01
- URL: https://github.blog/changelog/2026-06-01-evaluation-models-in-auto-for-individual-plans/

GitHub Copilot is now exposing evaluation models to individual non-enterprise users, and those models may be used by Copilot’s auto model selection. Users can opt out in settings.

Why it matters: This shows Copilot’s routing layer becoming more experimental and dynamic. For engineering leaders, developer experience, output stability, and reproducibility will depend increasingly on model selection policy rather than any single model.

Key takeaways:
- Auto model selection is no longer limited to a stable set of generally available models and may include evaluation-stage models.
- Teams that care about predictability, compliance, or regression analysis should define routing and opt-out expectations.

Tags: GitHub Copilot, model routing, evaluation models, AI governance
Scores: signal 6/5, novelty 6/5, actionability 6/5
