## Selected Items

## Updates to GitHub Copilot billing and plans
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-01
- URL: https://github.blog/changelog/2026-06-01-updates-to-github-copilot-billing-and-plans/

GitHub 宣布自 6 月 1 日起所有 Copilot 方案全面启用按 AI Credits 消耗计费；Copilot code review 除消耗 AI Credits 外，还会消耗 GitHub Actions minutes。同时新增用户级预算控制，并支持升级到 Copilot Max。

Why it matters: 这不是单纯定价新闻，而是 AI 编码能力从“席位制工具”转向“可计量基础设施消耗”的明确信号。对平台团队而言，预算、配额、Runner 规划、组织内归因和成本治理将成为 AI coding rollout 的核心工作。

Key takeaways:
- Copilot 的组织采纳将越来越像云资源治理：需要预算、额度、默认 runner 和使用策略。
- 如果团队启用 Copilot code review，需要把 Actions capacity 与 AI 成本一起纳入容量规划。

Tags: GitHub Copilot, AI billing, platform governance, developer tooling
Scores: signal 7/5, novelty 5/5, actionability 8/5

## Evaluation models in auto for individual plans
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-01
- URL: https://github.blog/changelog/2026-06-01-evaluation-models-in-auto-for-individual-plans/

GitHub Copilot 开始向个人非企业用户开放 evaluation models，这些模型也可能被纳入 Copilot 的 auto model selection。用户可以在设置中关闭通过自动模型选择使用 evaluation models。

Why it matters: 这反映出 Copilot 的模型路由层正在持续实验化和动态化。对工程负责人而言，这意味着开发体验、输出稳定性和可复现性会越来越依赖模型选择策略，而不仅是单一模型能力。

Key takeaways:
- 自动模型选择已不再只是在固定已发布模型之间切换，也可能纳入评估阶段模型。
- 如果团队关心可预测性、合规或回归排查，需要明确模型路由与 opt-out 策略。

Tags: GitHub Copilot, model routing, evaluation models, AI governance
Scores: signal 6/5, novelty 6/5, actionability 6/5

## How I AI: My First Impressions of Claude Opus 4.8 – Coding, Strategy, and Where It Shines
- Source: ChatPRD Blog
- Author: Claire Vo
- Publish date: 2026-06-01
- URL: https://www.chatprd.ai/how-i-ai/claude-opus-4-8-review

Why it matters: 可作为模型市场动向的弱信号参考，但不足以支撑技术决策、选型或评估体系设计。

Key takeaways:

- 缺少公开 harness、任务集、失败案例拆解，因此技术信号有限。
