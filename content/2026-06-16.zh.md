## Copilot usage metrics now include more of your active users
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-15
- URL: https://github.blog/changelog/2026-06-15-copilot-usage-metrics-now-include-more-of-your-active-users/

GitHub 更新了 Copilot 企业使用报表与 API，除了 IDE 等客户端上报的遥测外，也开始纳入服务端可确认的活动信号，因此此前因网络、代理、配置等原因未被客户端上报覆盖的活跃付费用户，现在会出现在单日和 28 天活跃报表中。

Why it matters: 这不是功能层面的新能力，而是组织采纳度量基础设施的修正。对平台团队和工程管理者而言，AI 编码工具的推广、续费、席位回收、团队对比和 ROI 分析，长期都依赖活跃用户数据；如果底层统计偏低，所有治理决策都会失真。该更新提升了 DAU 覆盖率，但也提醒团队：服务端补充信号无法提供与客户端同等丰富的 IDE、功能、模型和代码产出细节，因此高层采用率与细粒度行为分析仍需分开看待。

Key takeaways:
- 企业 Copilot 活跃用户统计将更接近真实使用情况，尤其在受代理、网络或客户端设置影响的环境中。
- 新增的服务端信号补足覆盖率，但不能完全替代客户端遥测的细粒度分析字段。
