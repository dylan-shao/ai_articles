## Staged publishing and new install-time controls for npm
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-22
- URL: https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm

GitHub announced two supply-chain security updates in npm CLI 11.15.0: staged publishing is now generally available, and new install-source control flags --allow-file, --allow-remote, and --allow-directory complement the existing --allow-git flag. The overall direction is finer-grained trust boundaries for both package publishing and installation.

Why it matters: While not directly about AI coding, this matters for teams using coding agents, automated dependency edits, and CI-driven builds. As agents increasingly modify package manifests, pull dependencies, and trigger release workflows, install-source restrictions and staged publishing help reduce the blast radius of malicious dependencies, accidental releases, and unintended propagation of internal packages.

Key takeaways:
- npm is making install provenance a first-class policy surface, which is useful for CI, locked-down build environments, and enterprise developer images.
- Staged publishing shifts package release from an all-at-once exposure model to a more controlled progressive rollout, which is valuable for high-risk or high-impact dependencies.
