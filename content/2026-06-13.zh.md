## Copilot code review: New configurations and controls - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-12
- URL: https://github.blog/changelog/2026-06-12-copilot-code-review-new-configurations-and-controls/

GitHub 为 Copilot code review 增加了三项关键企业控制：组织级 runner 类型配置、内容排除支持，以及仓库自定义指令字符限制取消。由于 Copilot code review 采用基于 GitHub Actions 的 agentic 架构，这些能力本质上是在给 AI 代码审查的执行环境、可见数据范围和行为约束提供集中化治理入口。

Why it matters: 这不是普通功能小更新，而是 AI 代码审查开始进入“平台治理”阶段的明确信号。对平台团队而言，真正难点不是让 agent 能跑，而是让它在合规、隔离、性能、成本和组织一致性约束下可规模化运行。组织级 runner 锁定意味着企业可以统一把 AI 审查流量导向自托管或大规格 runner；内容排除意味着可以更现实地处理敏感代码与合规边界；更长的自定义指令则让团队把审查规范、架构约束和语言风格固化为可执行策略。

Key takeaways:
- Copilot code review 的执行面与 GitHub Actions 深度绑定，runner 选择已成为 AI 审查架构设计的一部分。
- 组织级默认值与锁定能力，说明企业采用焦点正从“给开发者一个 AI 功能”转向“为整个组织定义受控默认行为”。
- 内容排除与更长指令窗口，使 AI 审查更接近可治理的内部平台能力，而非单点插件。

## GitHub Actions: Minimum version enforcement timeline for self-hosted runners - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-12
- URL: https://github.blog/changelog/2026-06-12-github-actions-minimum-version-enforcement-timeline-for-self-hosted-runners/

GitHub 恢复对自托管 Actions runner 最低版本的强制执行，并给出了时间线与兼容性要求。文章还披露了底层架构迁移背景：新的 Actions 基础设施已支撑每天超过 1.2 亿个 job，并显著提升企业作业启动吞吐。随着旧 runner 无法兼容新平台，升级 runner 版本将不只是 CI 维护事项，也会直接影响依赖 Actions 的 Copilot code review 和 cloud agent。

Why it matters: 对工程平台负责人来说，这条信息的价值在于它把“AI 开发工具可用性”与“CI/CD 基础设施卫生”连接起来了。既然 GitHub 把更多 agentic 能力放在 Actions 执行平面上，那么 runner 生命周期、版本漂移、网络出口、隔离等级与自动升级策略都会变成 AI 采用的关键阻塞项。很多团队会先采购 AI，再发现真正卡点是老旧 runner 池与平台治理缺口。

Key takeaways:
- 自托管 runner 版本管理正在从可靠性问题升级为 AI 工作流可用性问题。
- GitHub 明确展示了其将 agentic 开发体验建立在大规模 Actions 基础设施上的方向。
- 平台团队应尽快盘点 runner 版本、注册流程和升级自动化，避免 AI 代码审查或 agent 执行被基础设施门槛阻断。

## New Terms

- 组织级 runner 锁定：在组织层统一指定并强制 Copilot code review / cloud agent 使用某类 runner。
- 内容排除支持：为 AI 审查定义不可见或不参与处理的内容范围，以满足安全与合规要求。
- agentic architecture powered by GitHub Actions：AI 代码审查通过 Actions 作为执行平面运行，而非仅是静态模型调用。

## Implications

- 如果你在评估或推广 AI 代码审查，下一步不应只看模型效果，还要审查 runner 拓扑、网络隔离、日志策略和敏感内容边界。
- 平台团队需要把 Copilot/agent 能力纳入现有 Actions 平台治理，而不是把它当作独立 SaaS 功能采购。
- 更长的自定义指令空间意味着团队可以开始系统化沉淀审查 rubric、架构禁区和语言规范。
