## Copilot Memory has more controls for deletion, scope, and the Copilot CLI
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-26T21:05:28+00:00
- URL: https://github.blog/changelog/2026-05-26-copilot-memory-has-more-controls-for-deletion-scope-and-the-copilot-cli

GitHub updated Copilot Memory with better deletion guidance, a repository-level off switch, and additional memory controls in the Copilot CLI. The feature remains in public preview and is available to all paid Copilot plans.

Why it matters: For engineering orgs, long-term adoption of AI coding assistants depends on what gets remembered, where memory applies, and how it can be deleted or audited. This update is mostly control-plane work, but it hits core enterprise blockers: privacy, repo boundaries, team governance, and consistency in CLI-based workflows.

Key takeaways:
- Memory is evolving from a personalization feature into an enterprise capability that needs explicit boundaries, deletion semantics, and repo-level governance.
- Bringing memory controls into the CLI signals that AI assistant governance must extend beyond the IDE into terminal and automation workflows.
