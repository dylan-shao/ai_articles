## Quantifying infrastructure noise in agentic coding evals
- Source: Anthropic Engineering
- Author: N/A
- Publish date: N/A
- URL: https://www.anthropic.com/engineering/infrastructure-noise

Anthropic examines variance introduced by infrastructure configuration in agentic coding benchmarks, showing that on Terminal-Bench 2.0, resource setup alone can produce roughly a 6-point score swing—large enough to exceed typical leaderboard gaps between models. The post argues that unlike static benchmarks, agentic evals make the runtime part of the task itself, so CPU, RAM, time limits, container scheduling, dependency installation, and error handling all materially affect what is being measured.

Why it matters: This directly changes how enterprises should interpret coding-agent benchmarks. If runtime conditions are not tightly standardized, 'the better model' may simply be 'the model with more time, more memory, or a different sandbox policy.' For internal evals, vendor selection, and harness design, this methodological warning matters more than leaderboard deltas.

Key takeaways:
- Agentic coding evals are not pure model tests; the runtime environment is part of the measurement system and infra variables must be explicitly controlled and logged.
- Small leaderboard gaps may be meaningless when resource budgets, execution policies, and failure-recovery behavior differ across setups.
