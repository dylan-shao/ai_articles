## Improvements to secret scanning and public monitoring - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-15
- URL: https://github.blog/changelog/2026-07-15-improvements-to-secret-scanning-and-public-monitoring/

GitHub 发布了 secret scanning 和 public monitoring 的一组小幅增强：新增 Resend 与 APIclub 的 secret detector，VolcEngine secrets 默认启用 push protection，secret_scanning_alert webhook 增加 secret_category 字段，并在 public monitoring 告警列表中加入泄露归因、企业成员数和验证域名等洞察卡片。

Why it matters: 这不是 AI 编码主题的核心进展，但对负责开发平台、安全治理和自动化响应的团队有实际价值。尤其是 webhook 分类字段可直接改善告警路由、优先级策略和后续自动修复流程设计。

Key takeaways:
- secret_scanning_alert webhook 新增 secret_category，可区分 default 与 generic 类型，便于下游系统按风险等级和可信度分流。
- public monitoring 页面新增洞察卡片，说明 GitHub 正把秘密泄露检测从“发现问题”推进到“支持治理与归因分析”。
- 默认 push protection 覆盖更多 secret 类型，意味着平台安全基线正逐步从告警转向提交前阻断。
