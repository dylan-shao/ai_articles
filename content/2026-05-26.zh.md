## Staged publishing and new install-time controls for npm
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-22
- URL: https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm

GitHub 宣布 npm CLI 11.15.0 提供两项面向供应链安全的更新：staged publishing 正式可用，以及新增安装来源控制参数 --allow-file、--allow-remote、--allow-directory，用于补充已有的 --allow-git。整体方向是把包发布与安装阶段的信任边界做得更细。

Why it matters: 虽然这不是 AI coding 直接进展，但对使用代理编程、自动依赖修改、CI 自动构建的团队很相关。随着代码代理越来越频繁地生成 package.json 变更、拉取依赖并触发发布流程，安装来源限制和分阶段发布可以降低恶意依赖、错误发布和内部包扩散风险。

Key takeaways:
- npm 开始把“能从哪里安装”变成显式策略，适合在 CI、受控构建环境和企业开发镜像中启用。
- staged publishing 让包发布从一次性全量暴露，转向更可控的渐进式发布流程，适合高风险或高影响依赖。
