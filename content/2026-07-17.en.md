## REST API endpoints for Visual Studio Subscription management - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-16
- URL: https://github.blog/changelog/2026-07-16-rest-api-endpoints-for-visual-studio-subscription-management/

GitHub Enterprise Cloud added REST APIs for Visual Studio Subscription assignment management, including listing enterprise assignments, bulk mapping VSS UPNs to GitHub handles, and deleting manual matches. The update mainly addresses cases where UPN formats do not align with SCIM identities and auto-matching fails.

Why it matters: This is not directly about AI coding, but it is operationally meaningful for platform teams: it turns a brittle manual license-matching process into a scriptable, auditable identity-governance workflow that can be integrated with enterprise IAM, SCIM, and joiner/mover/leaver automation.

Key takeaways:
- New GET/PUT/DELETE endpoints allow bulk management of VSS-to-GitHub user mappings.
- Especially useful in enterprises with identity-format mismatches, reducing manual UI work and assignment errors.

Tags: GitHub Enterprise, identity governance, REST API, platform operations

## Xcode 27 runner image now in public preview - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-16
- URL: https://github.blog/changelog/2026-07-16-xcode-27-runner-image-now-in-public-preview/

GitHub Actions introduced a public preview of hosted macOS runners for Xcode 27 and shifted its macOS image support model to be organized by major Xcode version rather than underlying OS version. New labels include xcode-27 and xcode-27-xlarge, with support limited to arm64 macOS runners.

Why it matters: For teams maintaining Apple build pipelines, this reduces uncertainty from toolchain drift. Modeling runners by major Xcode version helps align CI environments more explicitly with local development, release validation, and compatibility testing.

Key takeaways:
- Runner labels now map more directly to toolchain versions instead of implicitly depending on macOS image lineage.
- Arm64-only support means legacy pipelines still relying on Intel runners need migration planning.

Tags: GitHub Actions, CI/CD, Apple toolchain, runner images
Scores: signal 6/5, novelty 6/5, actionability 7/5

## Advanced search for Projects is generally available - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-16
- URL: https://github.blog/changelog/2026-07-16-advanced-search-for-projects-is-generally-available/

GitHub Projects filter bars now generally support advanced search with AND/OR expressions, and add a new reviews: filter for pull-request review state. The update also introduces a 90-day retention policy for deployment statuses, removing older statuses from REST and GraphQL APIs.

Why it matters: This matters modestly for engineering management and platform observability: the search additions improve project-level triage views, while the 90-day deployment-status retention change can quietly break internal reporting, governance scripts, or API integrations that rely on long-lived history.

Key takeaways:
- Projects views are more expressive, reducing the need to maintain many static views for different questions.
- The 90-day deployment-status retention policy is the more consequential change; downstream consumers should audit historical dependencies.

Tags: GitHub Projects, search, developer workflow, API retention
Scores: signal 5/5, novelty 5/5, actionability 6/5
