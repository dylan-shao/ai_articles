## Create issues from Slack with Copilot
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-03-30
- URL: https://github.blog/changelog/2026-03-30-create-issues-from-slack-with-copilot/

GitHub added Copilot-powered issue creation to its Slack integration. Users can @GitHub in a Slack channel, describe work in natural language, and generate structured GitHub Issues with titles, bodies, assignees, labels, milestones, and parent/child issue hierarchy. Teams can refine the issue in-thread and set channel-level default repositories.

Why it matters: For platform and engineering leads, this shows AI coding agents evolving from code-generation assistants into workflow entry-point agents. Requirement capture no longer depends on humans manually translating chat into tickets; work can be structured directly in the collaboration context. That has implications for intake throughput, issue quality, project traceability, and eventual closed-loop flows where agents pick up tickets and produce code changes.

Key takeaways:
- Natural-language-to-structured-issue conversion is moving into Slack, making intake friendlier for cross-functional teams.
- Support for sub-issue hierarchy and in-thread refinement shows GitHub embedding agent behavior into project management, not just code editing.
- Channel-level default repository settings reduce friction in multi-repo organizations, indicating a deployable workflow feature rather than a demo-only capability.
