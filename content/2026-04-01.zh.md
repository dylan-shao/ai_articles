## Quantifying infrastructure noise in agentic coding evals
- Source: Anthropic Engineering
- Author: N/A
- Publish date: N/A
- URL: https://www.anthropic.com/engineering/infrastructure-noise

Anthropic 分析了 agentic coding benchmark 中由基础设施配置引入的方差，指出在 Terminal-Bench 2.0 上，仅资源配置差异就可带来约 6 个百分点的成绩波动，而且这种波动足以超过模型之间常见的 leaderboard 差距。文章强调，agentic eval 与静态基准不同，运行时环境本身是问题求解的一部分，因此 CPU、RAM、超时、容器调度、依赖安装和错误恢复机制都会改变“被测对象”。

Why it matters: 这直接影响企业如何解读 coding-agent 评测结果。若运行环境没有严格标准化，那么“模型更强”可能只是“给了更多时间、更多内存、不同的 sandbox 策略”。对于内部基准、采购评估和 agent harness 设计，这篇文章提供了比单纯看 leaderboard 更重要的方法论提醒。

Key takeaways:
- Agentic coding eval 不是纯模型测试，运行环境是测量系统的一部分；infra 变量必须被显式控制和记录。
- 小幅 leaderboard 差距可能没有统计意义，尤其当资源预算、执行策略和失败恢复路径不一致时。

## Copilot usage metrics now includes organization-level GitHub Copilot CLI activity
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-03-17T00:00:00+00:00
- URL: https://github.blog/2026/

GitHub 在企业级 Copilot 指标中纳入了 organization-level 的 Copilot CLI 活动，可用于更完整地观测团队在终端环境中的 AI 使用情况。候选信息来自汇总式 changelog 页面，技术细节有限，但它表明 GitHub 正在把 AI 编码采用度量从 IDE 扩展到 CLI 与更广泛的 agent surface。

Why it matters: 这对平台团队有一定参考价值，因为组织采用 AI coding 之后，真正难的是治理和度量，而不是开通许可证。CLI 指标意味着企业开始能把命令行场景纳入使用分析、ROI 评估和合规审计。

Key takeaways:
- AI coding 采用度量正在从编辑器扩展到终端与 agent workflow。
- 对企业而言，观测层建设正成为 AI coding 规模化落地的必要条件。
