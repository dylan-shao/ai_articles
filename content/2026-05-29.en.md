## CodeQL 2.25.5 improves query accuracy for GitHub Actions - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-28
- URL: https://github.blog/changelog/2026-05-28-codeql-2-25-5-improves-query-accuracy-for-github-actions/

CodeQL 2.25.5 ships several precision improvements across GitHub Actions, Java/Kotlin, and C/C++. The most relevant change is expanded sink modeling for GitHub Actions, including scripts executed via Python modules and go run in directories; actions/unpinned-tag now also analyzes composite action metadata files. On Java/Kotlin, a new path-injection[read] sink kind helps distinguish read-only path sinks and reduces false positives in queries such as ZipSlip.

Why it matters: For platform and security engineering teams, modeling-layer improvements matter more than surface-level feature launches: they determine whether scan results are trustworthy enough for default enforcement and whether GitHub Actions supply-chain risks are covered comprehensively. Lower false positives often directly determine whether developers keep scanners enabled.

Key takeaways:
- GitHub Actions analysis now extends beyond workflow files into composite action metadata and additional execution sinks, broadening static coverage of CI attack surface.
- The new read-only path sink category gives Java/Kotlin analysis finer-grained risk semantics, helping reduce false positives and making rules more enforceable in practice.

Tags: CodeQL, GitHub Actions, Static Analysis, DevSecOps, CI Security

## Claude Opus 4.8 is generally available for GitHub Copilot - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-28
- URL: https://github.blog/changelog/2026-05-28-claude-opus-4-8-is-generally-available-for-github-copilot/

Claude Opus 4.8 is now generally available in GitHub Copilot across multiple surfaces including VS Code, Visual Studio, Copilot CLI, cloud agent, JetBrains, and Xcode. GitHub claims improved code understanding, generation, complex reasoning, and large-codebase navigation versus prior models, but the post provides no public benchmark details. The model launches with a 15x premium request multiplier, and enterprise admins must explicitly enable the policy.

Why it matters: The main value here is organizational adoption: model selection is shifting from a single default to task- and cost-aware routing. For engineering leaders, the key issue is not the launch itself, but how to build model admission, budget control, and task allocation policies.

Key takeaways:
- GitHub Copilot is becoming increasingly multi-model across chat, edit, agent, and CLI surfaces, indicating model routing will become a normal part of developer workflows.
- The 15x premium multiplier is a reminder that model evaluation must be tied to cost governance; otherwise premium models can inflate spend without proportional productivity gains.

Tags: GitHub Copilot, Claude Opus, Model Routing, AI Coding, Enterprise Adoption

## Hard budget limits now available for GitHub Advanced Security - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-28
- URL: https://github.blog/changelog/2026-05-28-hard-budget-limits-now-available-for-github-advanced-security/

GitHub Advanced Security now supports hard budget limits. Enterprise admins can set an enforceable license-count cap; once reached, new license assignments are blocked and GHAS will not be enabled on additional repositories. Previously, budgets were soft notifications only at 75/90/100% thresholds, so this change turns budget control from alerting into enforcement.

Why it matters: This is not an AI-coding technical update, but it matters for platform governance: both security and AI tooling are moving toward seat- plus usage-based cost structures, and organizations need enforceable guardrails to prevent uncontrolled expansion from automated provisioning and group sync.

Key takeaways:
- The move from soft budgets to hard budgets signals GitHub is adding governance primitives better suited to large organizations.
- If your org is rolling out Copilot, GHAS, or other seat/usage-priced developer tools, features like this belong in a unified FinOps/PlatOps control plane.

Tags: GitHub Advanced Security, Platform Governance, FinOps, Developer Tooling
Scores: signal 5/5, novelty 4/5, actionability 7/5
