## Copilot Memory supports user preferences for Pro, Pro+ users - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-15
- URL: https://github.blog/changelog/2026-05-15-copilot-memory-supports-user-preferences-for-pro-pro-users/

GitHub 将 Copilot Memory 从仅支持仓库级记忆，扩展到用户级偏好记忆。系统可保存用户显式声明或隐式推断的个人偏好，并在未来跨仓库、跨 Copilot 体验复用，例如提交风格、PR 结构和沟通语气。

Why it matters: 这比普通“个性化”更重要：它意味着编码代理开始形成长期、跨上下文的个人工作模型，而不是每次会话都重新学习。对工程组织而言，这会直接影响提示工程、代码审查一致性、代理权限边界、隐私治理与团队标准如何和个人偏好共存。

Key takeaways:
- 记忆层从“仓库上下文”扩展到“用户画像”，说明代理正在向长期个体适配演进。
- 用户可查看和删除个人记忆，表明产品已开始正视可控性与隐私治理问题。

Tags: copilot, agent-memory, developer-tooling, personalization
Scores: signal 8/5, novelty 8/5, actionability 7/5

## Grok Code Fast 1 deprecated - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-15
- URL: https://github.blog/changelog/2026-05-15-grok-code-fast-1-deprecated/

GitHub 已在所有 Copilot 体验中弃用 Grok Code Fast 1，并建议迁移到 GPT-5 mini 或 Claude Haiku 4.5。企业管理员需通过模型策略确认替代模型可用，工作流与集成也需要同步更新。

Why it matters: 这不是单纯的模型下线通知，而是提醒组织：AI 编码平台的供应层是动态变化的。若你的 IDE、CLI、代码审查或 agent 流程绑定特定模型，就需要建立模型替换、兼容性测试和管理员策略变更的治理机制。

Key takeaways:
- 编码代理栈中的底层模型不是稳定依赖项，企业需要把模型替换纳入变更管理。
- 管理员模型策略已成为可用性与团队体验的关键控制面。

Tags: copilot, model-governance, enterprise-adoption, change-management
Scores: signal 6/5, novelty 5/5, actionability 8/5

## Timestamp fields in GitHub Projects - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-15
- URL: https://github.blog/changelog/2026-05-15-timestamp-fields-in-github-projects/

GitHub Projects 新增内建时间戳字段 Created、Updated、Closed，可用于排序和筛选 issue、draft issue 与 pull request。Updated 还会反映项目字段变更，因此同一条目在不同项目中的时间戳可能不同。

Why it matters: 单看功能不大，但对平台团队与工程效能团队有用：这些字段能更方便地构建“最近完成”“卡住未更新”“状态刚变化”的视图，也为 agent 驱动的项目自动化和流程度量提供更清晰的数据基础。

Key takeaways:
- Projects 正在变成更适合自动化与流程分析的数据面板，而不只是手工看板。
- Updated 字段包含项目字段变更，适合用来监控流程流转，而不仅是代码对象本身变化。

Tags: github-projects, workflow-automation, engineering-metrics
Scores: signal 5/5, novelty 4/5, actionability 6/5
