## REST API endpoints for Visual Studio Subscription management - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-16
- URL: https://github.blog/changelog/2026-07-16-rest-api-endpoints-for-visual-studio-subscription-management/

GitHub Enterprise Cloud 新增用于管理 Visual Studio Subscription 分配的 REST API，支持查询企业级分配、批量将 VSS UPN 映射到 GitHub 账号，以及删除手动匹配关系。它主要解决 UPN 格式与 SCIM 身份不一致时无法自动匹配、只能在 UI 中手工修复的问题。

Why it matters: 这不是 AI coding 本身，但对大型工程平台团队有直接操作价值：它把原本脆弱的手工许可匹配流程转成可脚本化、可审计的身份治理工作流，适合与企业 IAM、SCIM 以及开发者入离职自动化集成。

Key takeaways:
- 新增 GET/PUT/DELETE 接口，允许批量管理 VSS 与 GitHub 用户的映射关系。
- 对身份格式不一致的企业环境尤其有用，可减少 UI 手工修复和错配风险。

Tags: GitHub Enterprise, identity governance, REST API, platform operations
Scores: signal 6/5, novelty 5/5, actionability 8/5

## Xcode 27 runner image now in public preview - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-16
- URL: https://github.blog/changelog/2026-07-16-xcode-27-runner-image-now-in-public-preview/

GitHub Actions 提供 Xcode 27 的托管 macOS runner 预览版，并将 macOS 镜像支持模型改为按 Xcode 主版本划分，而不是按底层操作系统划分。新标签包括 xcode-27 和 xcode-27-xlarge，仅支持 arm64 macOS runner。

Why it matters: 对维护 Apple 构建流水线的平台团队来说，这个变化降低了 toolchain 漂移带来的不确定性。按 Xcode 主版本建模 runner，有助于把 CI 环境与本地开发、发布验证和兼容性测试更明确地对齐。

Key takeaways:
- runner 标签现在更直接表达工具链版本，而不是隐式依赖 macOS 镜像。
- 仅支持 arm64，意味着仍依赖 Intel runner 的遗留流水线需要提前迁移评估。

Tags: GitHub Actions, CI/CD, Apple toolchain, runner images
Scores: signal 6/5, novelty 6/5, actionability 7/5

## Advanced search for Projects is generally available - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-16
- URL: https://github.blog/changelog/2026-07-16-advanced-search-for-projects-is-generally-available/

GitHub Projects 的过滤栏现已 GA 支持高级搜索，可用 AND/OR 构建更复杂的视图；同时新增 reviews: 过滤器以按 PR review state 检索项目项。更新还宣布 deployment statuses 采用 90 天保留策略，超期状态将从 REST 和 GraphQL API 中删除。

Why it matters: 这对工程管理和平台可观测性有一定影响：前者提升了项目级 triage 与执行面板的表达能力，后者则可能悄悄影响依赖 deployment status 历史数据的内部报表、治理脚本或 API 集成。

Key takeaways:
- Projects 视图表达能力增强，可减少为不同问题维护大量静态视图的成本。
- deployment status 的 90 天保留策略更值得注意，相关数据消费方应检查历史依赖。

Tags: GitHub Projects, search, developer workflow, API retention
Scores: signal 5/5, novelty 5/5, actionability 6/5
