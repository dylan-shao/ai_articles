## IP allow list coverage for EMU namespaces in general availability - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-08
- URL: https://github.blog/changelog/2026-06-08-ip-allow-list-coverage-for-emu-namespaces-in-general-availability/

GitHub announced GA for IP allow list coverage on Enterprise Managed Users (EMU) namespaces. Enterprises can now apply native IP allow list policy consistently to repository access within EMU user namespaces, covering the web UI, Git protocol, APIs, and credentials such as PATs, app tokens, and SSH keys.

Why it matters: While not an AI-coding capability itself, this matters for enterprise rollout of Copilot, coding agents, and automated developer workflows: as more development activity happens through APIs, CLIs, agents, and service credentials, network boundaries and credential usage need centralized enforcement. This closes a governance gap around user-owned EMU namespaces.

Key takeaways:
- EMU user namespaces can now be governed by enterprise-wide IP allow lists rather than leaving user-owned areas as exceptions.
- Coverage spans web, Git, APIs, and multiple credential types, bringing agentic development and automation access paths under the same network policy.
