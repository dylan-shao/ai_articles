## View and manage agent sessions from issues and projects
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-23
- URL: https://github.blog/changelog/2026-04-23-view-and-manage-agent-sessions-from-issues-and-projects/

GitHub now lets users view, open, and steer Copilot cloud agent sessions directly from Issues and Projects. New UI elements include an issue header session pill, sidebar session panels, and agent sessions enabled by default in project views, bringing agent execution into the primary work-tracking surfaces.

Why it matters: This is more than a UI tweak: it embeds agent execution into the system of record for software delivery. For engineering organizations, that means the agent is no longer just a private IDE assistant but a team-visible, steerable actor, which is foundational for task delegation, auditing, recovery, and cross-role collaboration.

Key takeaways:
- Agent sessions now live in issues/projects, signaling GitHub’s shift from personal assistant UX to team workflow objects.
- The phrase 'steer the agent' is the key signal: future platform design will emphasize human intervention, constraints, feedback loops, and session-state management.

## Copilot Chat improvements for pull requests
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-23
- URL: https://github.blog/changelog/2026-04-23-copilot-chat-improvements-for-pull-requests/

GitHub expanded Copilot Chat for PR workflows: when a pull request is provided as context, chat now incorporates comments, changed files, commits, and reviews; it also adds structured PR review and summary capabilities, including contextual questioning directly from diffs.

Why it matters: PRs are the highest-value surface for real AI coding adoption because they combine code, context, review feedback, and intent. GitHub is moving chat from code explanation toward change understanding, review generation, and context compression—capabilities that directly affect reviewer throughput, change comprehension, and cross-team coordination.

Key takeaways:
- High-quality PR AI depends on multi-source context assembly, not just the diff; comments, commits, and review history are now part of the default input.
- Structured review/summarization is becoming a standard interaction pattern, implying future evaluations should measure change understanding and review quality, not only code generation.

## Copilot cloud agent fields added to usage metrics
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-23
- URL: https://github.blog/changelog/2026-04-23-copilot-cloud-agent-fields-added-to-usage-metrics/

GitHub added a `used_copilot_cloud_agent` field to the Copilot usage metrics API for enterprise/org user reports across both daily and 28-day windows. It mirrors `used_copilot_coding_agent` and will coexist until 2026-08-01, allowing dashboards and integrations to migrate without breaking changes.

Why it matters: This may look like a simple rename, but the bigger signal is that cloud agent usage is now something to measure, report, and govern explicitly. Organizational AI coding adoption is shifting from seat allocation to capability-level instrumentation, which matters for ROI tracking, policy control, and phased rollout.

Key takeaways:
- Organizations should separate agent usage from traditional Copilot metrics rather than collapsing all interaction modes into one adoption number.
- The coexistence window until August creates a practical migration period for BI pipelines, cost analytics, and governance dashboards.

## New Terms

- Copilot cloud agent: GitHub’s new name for the former Copilot coding agent, emphasizing cloud-executed agent workflows.
- Agent session pill: A UI entry point in issue headers that aggregates active and completed agent sessions.
- Steer the agent: An interaction model where humans guide, correct, or augment an agent while it is executing.
- PR review / PR summary as chat abilities: Treating structured review and change summarization as first-class chat capabilities rather than ad hoc prompting.
