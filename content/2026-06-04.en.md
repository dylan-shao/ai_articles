- Source: GitHub Changelog

The important signal in this monthly release is not individual feature churn, but the productization of an agent execution surface inside VS Code/Copilot: the Agents window is now in Stable preview for task-oriented, multi-project work; remote agents can run over SSH or Dev Tunnels and continue when the client disconnects; BYOK now extends to air-gapped environments with model controls for utility tasks; and GitHub is continuing investment in the Agent Host Protocol for synchronizing agent session state across clients.

Why it matters: This shows AI coding in the IDE moving from a local interactive assistant to a managed, remote-capable, stateful, and governable agent system. For platform teams, AHP, remote sessions, and BYOK/isolation features matter more than model swaps because they determine whether agents can operate inside enterprise networks, controlled execution environments, and existing engineering governance.

Key takeaways:
- The Agents window consolidates task execution, multi-project navigation, and change review into an agent-first workspace, signaling IDE redesign around agent workflows.
- Remote agents plus disconnected session continuity point toward long-running tasks, background execution, and a model closer to CI/dev container execution than local chat.
- BYOK in air-gapped environments with per-task model routing reflects rising enterprise demand for compliance, cost control, and model governance.
- Agent Host Protocol is the most important lower-level signal to track: it could become the interface layer for sharing agent session state across clients and hosts.

## New Terms

- Agent Host Protocol (AHP): an open protocol direction for synchronizing agent session state across multiple clients.
- BYOK in air-gapped environments: an enterprise deployment mode where teams use their own model keys inside isolated networks.
- Remote agents: coding agent sessions that run persistently on remote machines over SSH or Dev Tunnels.

## Themes

- AI coding is moving from chat-style assistance to task-oriented, long-running, remote-executed agent runtimes.
- Enterprise adoption constraints are shifting from raw model quality to protocol design, isolation, session persistence, and governance.
- The IDE, remote host, and Git workflow are being fused into a unified agent execution surface.

## Implications

- If you are building an internal coding-agent platform, prioritize session state architecture, remote execution hosts, disconnect recovery, and Git/code-review integration over model leaderboard comparisons.
- Model routing and permission boundaries need to be defined early: which tasks may use external models, and which must run in internal or sandboxed environments.
- Track whether AHP or similar protocols mature openly, since that will shape the feasibility of a unified agent experience across IDE, CLI, and web surfaces.
