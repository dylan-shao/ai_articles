## C++ code intelligence for GitHub Copilot CLI in public preview
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-22
- URL: https://github.blog/changelog/2026-04-22-c-code-intelligence-for-github-copilot-cli-in-public-preview/

GitHub introduced public-preview C++ code intelligence for Copilot CLI via the Microsoft C++ Language Server, extending IDE-grade semantic understanding into command-line agent workflows. Instead of relying only on grep-style search, Copilot can use symbol definitions, references, call hierarchies, and type information, with accuracy depending on compile_commands.json and project setup.

Why it matters: This shows command-line coding agents are evolving from 'string search plus reasoning' toward 'language-server-backed semantic navigation.' For teams maintaining large C++ or systems codebases, this infrastructure directly affects whether an agent can reliably locate context amid macros, templates, and build-specific configuration; it also suggests future evaluations should separate text-search ability from semantic understanding.

Key takeaways:
- Copilot CLI is now explicitly integrating language-server capabilities rather than depending only on grep/full-text search.
- For complex languages and large repositories, build artifacts and indexing configuration are becoming prerequisites for agent quality.

## Bring your own language model key in VS Code now available
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-22
- URL: https://github.blog/changelog/2026-04-22-bring-your-own-language-model-key-in-vs-code-now-available/

GitHub added BYOK support in VS Code for Copilot Business and Enterprise users, allowing organizations to reuse their own API keys for Anthropic, Gemini, OpenAI, OpenRouter, Azure, and local models via Ollama and Foundry Local. These BYOK models are available in VS Code Chat, the built-in plan agent, and custom agents, but not for code completions, with billing handled directly by the chosen provider.

Why it matters: This is more than a model catalog expansion; it signals a shift in enterprise AI coding architecture, where model routing, cost ownership, compliance boundaries, and local-model access are moving into the IDE layer. For platform teams, that means revisiting default model policies, audit boundaries, agent compatibility, and how different models are assigned across agentic and human-in-the-loop workflows.

Key takeaways:
- Enterprises can now plug self-managed and local models directly into the GitHub Copilot experience, reducing dependence on a single hosted model path.
- BYOK currently covers chat and agents but not code completion, indicating separate governance layers for different coding-assistance modes.

Tags: BYOK, VS Code, enterprise adoption, model routing, developer tooling
Scores: signal 7/5, novelty 7/5, actionability 8/5

## GitHub CLI: Opt-out usage telemetry
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-22
- URL: https://github.blog/changelog/2026-04-22-github-cli-opt-out-usage-telemetry/

Starting in v2.91.0, GitHub CLI sends pseudonymous usage telemetry by default and provides both opt-out and log modes. Users can inspect the open-source telemetry implementation and preview the exact JSON payload for a given command via GH_TELEMETRY=log or gh config set telemetry log without actually sending it.

Why it matters: Telemetry is increasingly critical for developer tools and AI coding infrastructure because product teams need to know which commands, agent flows, and context-entry points are actually used. But default collection also directly hits enterprise privacy and compliance boundaries. GitHub's combination of 'auditable implementation + local payload preview + opt-out' offers a relatively mature governance pattern for developer tooling.

Key takeaways:
- Telemetry design for developer tools is starting to emphasize verifiability and explainability, not just an on/off switch.
- Platform teams can borrow this pattern when designing audit-friendly data collection for internal agents and CLIs.

Tags: telemetry, GitHub CLI, governance, developer tooling, privacy
