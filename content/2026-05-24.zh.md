## Staged publishing and new install-time controls for npm
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-22
- URL: https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm

GitHub 宣布 npm CLI 11.15.0 提供两项供应链安全能力：staged publishing 正式可用，以及新增更细粒度的安装来源白名单参数，包括 --allow-file、--allow-remote、--allow-directory，并与既有 --allow-git 配合使用。

Why it matters: 对负责开发平台、构建系统和内部包生态的团队来说，这类控制能把“依赖可以从哪里安装”变成可策略化的约束，降低 CI、agent 执行环境和自动化代码修改流程中被意外拉取本地/远程内容的风险。staged publishing 也适合高影响包的渐进发布与回滚治理。

Key takeaways:
- npm 现在支持更细粒度的安装来源控制，可在自动化环境中限制 file、remote、directory、git 等来源。
- staged publishing 有助于在关键包发布中实施渐进式发布、观测与回滚，降低供应链和运维风险。

## New Terms

- staged publishing：先分阶段向部分用户或范围发布包，再逐步扩大发布面的机制。
- install-source allowlist flags：通过命令行参数限制依赖安装来源类型的机制。

## Themes

- 开发工具链的默认安全边界正在变得更细粒度。
- 包管理器开始提供更适合企业治理与自动化执行环境的策略控制。

## Implications

- 如果你的 CI、代码代理或内部构建系统会执行 npm install，应评估是否默认禁用不必要的来源类型，并只对白名单场景放开。
- 对高价值内部包，可以把 staged publishing 纳入发布流水线，结合监控和回滚门禁。
