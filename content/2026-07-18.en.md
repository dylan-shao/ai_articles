## Copilot code review: Customization and configurability improvements
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-17
- URL: https://github.blog/changelog/2026-07-17-copilot-code-review-customization-and-configurability-improvements/

GitHub expanded Copilot code review configurability: review instructions are now read from the PR head branch, enabling branch-local testing and validation; it also recognizes REVIEW.md, GEMINI.md, and CLAUDE.md, and adds custom setup steps, firewall support, and independent runner configuration.

Why it matters: This is a meaningful step from AI code review being merely available to being governable, iteratable, and enterprise-deployable. Platform teams can now manage review instructions more like CI policy and validate them before merge, while firewall and runner support reduce friction in constrained networks, regulated environments, and customized build contexts.

Key takeaways:
- Reading review instructions from the head branch enables prompt/policy experimentation and regression validation inside the PR itself.
- Support for more convention files and enterprise controls suggests GitHub is turning the review agent into infrastructure that fits existing engineering governance.

## Repository-level GitHub Copilot usage metrics generally available
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-17
- URL: https://github.blog/changelog/2026-07-17-repository-level-github-copilot-usage-metrics-generally-available/

The GitHub Copilot usage metrics API now exposes repo-level daily reports via new enterprise and org repos-1-day endpoints, showing per-repository PRs created/merged by the Copilot coding agent and code review activity with suggestion counts by comment type.

Why it matters: This fills a major observability gap in AI coding adoption by moving from user/org-level metrics down to code-asset-level visibility. For engineering effectiveness leaders, AI usage is no longer just about seat utilization; it can now be correlated with repository health, stack, service criticality, team maturity, and review throughput to drive more targeted rollout, enablement, and ROI attribution.

Key takeaways:
- The new repos-1-day API enables per-repo analysis of Copilot coding agent and code review activity.
- This creates a data foundation for AI-readiness scoring, priority repo enablement, pilot evaluation, and internal metrics programs.

Tags: copilot-metrics, repository-analytics, engineering-effectiveness, enterprise-adoption, governance
Scores: signal 9/5, novelty 7/5, actionability 10/5

## GitHub Copilot app now available in the usage metrics API
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-07-17
- URL: https://github.blog/changelog/2026-07-17-github-copilot-app-now-available-in-the-usage-metrics-api/

GitHub added Copilot app usage to the enterprise and org usage metrics API, with new daily_active_copilot_app_users and totals_by_copilot_app fields in 1-day and 28-day reports covering sessions, requests, prompts, and token usage.

Why it matters: While less strategic than repo-level reporting, this closes an important cross-surface measurement gap. Many organizations are now using IDE, chat, review, coding agents, and standalone app experiences in parallel; without unified visibility into activity and token consumption across these surfaces, budgeting, governance, and workflow optimization stay incomplete.

Key takeaways:
- The Copilot app is now included in the existing metrics API, making it easier to unify multi-surface usage in enterprise BI or FinOps dashboards.
- Token and request volume can help platform teams identify high-consumption surfaces and revisit default entry-point strategy.
