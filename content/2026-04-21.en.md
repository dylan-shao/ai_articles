## Changes to GitHub Copilot plans for individuals - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-20
- URL: https://github.blog/changelog/2026-04-20-changes-to-github-copilot-plans-for-individuals/

GitHub announced a pause on new signups for Copilot Pro, Pro+, and Student, tighter usage caps for individual plans, and removal of Opus models from Pro while keeping limited availability in higher tiers.

Why it matters: This is less a product launch than a capacity and cost-governance signal. For engineering leaders, it shows mainstream AI coding products moving into stricter quota tiering, model segmentation, and reliability-first operations, which should influence procurement, defaults, and expectation-setting.

Key takeaways:
- Paused signups and tighter caps suggest demand, inference cost, or capacity pressure is materially shaping Copilot product policy.
- Higher-end models are being pushed into premium tiers, making model access itself a central monetization and resource-governance lever.
- Teams relying on individual subscriptions instead of enterprise agreements may face more volatility in availability, budgeting, and quota management.

Tags: AI coding, Copilot, pricing, capacity management, developer tooling

## Sunsetting SHA-1 in HTTPS on GitHub - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-20
- URL: https://github.blog/changelog/2026-04-20-sunsetting-sha-1-in-https-on-github/

GitHub will run a brownout in July 2026 and fully disable SHA-1 in HTTPS/TLS in September 2026, affecting browsers, Git over HTTPS, API clients, and older software in dependent chains.

Why it matters: While not directly about AI coding, it has clear operational value for platform teams: many enterprises still have outdated Git clients, old build images, embedded TLS stacks, or proxy chains that can quietly break CI/CD, fetch/push flows, and automation on the cutover date.

Key takeaways:
- Audit every browser, CLI, CI image, SDK, and proxy that reaches GitHub over HTTPS.
- The brownout window provides a practical checkpoint for compatibility drills and alert validation.
- This is a good forcing function to remove legacy crypto dependencies and refresh developer base images.

Tags: GitHub, TLS, platform engineering, legacy clients, CI/CD
Scores: signal 5/5, novelty 3/5, actionability 8/5
