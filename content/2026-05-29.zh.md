## CodeQL 2.25.5 improves query accuracy for GitHub Actions - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-28
- URL: https://github.blog/changelog/2026-05-28-codeql-2-25-5-improves-query-accuracy-for-github-actions/

CodeQL 2.25.5 发布了多项精度改进，重点覆盖 GitHub Actions、Java/Kotlin 和 C/C++。其中最有价值的是扩展了 GitHub Actions 的 sink 建模，新增对通过 Python 模块执行脚本以及在目录中使用 go run 的检测；同时 actions/unpinned-tag 现在也分析 composite action 的 action.yml/action.yaml。Java/Kotlin 方面引入 path-injection[read] 这类只读路径 sink，并据此降低 ZipSlip 等规则的误报。

Why it matters: 对平台与安全工程团队来说，这类“规则建模层”的改进比普通功能发布更重要：它决定了扫描结果是否可信、是否能进入默认门禁，以及 GitHub Actions 供应链风险能否被更完整地覆盖。误报下降通常直接决定开发者是否继续保留扫描。

Key takeaways:
- GitHub Actions 查询覆盖面扩大到 composite action 元数据，以及更多脚本执行 sink，意味着 workflow 之外的风险面也开始被静态分析纳入。
- 通过引入 read-only path sink，CodeQL 在 Java/Kotlin 路径注入与 ZipSlip 场景中更细粒度地区分风险强弱，有助于降低误报并提升规则可执行性。

Tags: CodeQL, GitHub Actions, Static Analysis, DevSecOps, CI Security

## Claude Opus 4.8 is generally available for GitHub Copilot - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-28
- URL: https://github.blog/changelog/2026-05-28-claude-opus-4-8-is-generally-available-for-github-copilot/

Claude Opus 4.8 已在 GitHub Copilot 中正式可用，可在 VS Code、Visual Studio、Copilot CLI、云端 agent、JetBrains、Xcode 等多个入口选择。GitHub 声称该模型在代码理解、生成、复杂问题求解和大代码库导航上优于前代，但本文未给出公开评测数据。当前该模型以 15X premium request multiplier 计费，且企业管理员需要显式开启策略。

Why it matters: 这条更新的价值主要在组织采用层面：模型选择正在从“单一默认模型”转向“按任务和成本分层路由”。对工程管理者而言，关键不是新模型上线本身，而是如何建立模型准入、预算控制与任务分配策略。

Key takeaways:
- GitHub Copilot 的多模型化继续深化，且覆盖 chat/edit/agent/CLI 等多个执行面，说明实际工作流中的模型路由会越来越常见。
- 15X 的 premium multiplier 提醒团队必须把模型效果评测与成本治理绑定，否则高端模型很容易只带来账单增长而非生产力提升。

Tags: GitHub Copilot, Claude Opus, Model Routing, AI Coding, Enterprise Adoption
Scores: signal 6/5, novelty 5/5, actionability 6/5

## Hard budget limits now available for GitHub Advanced Security - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-05-28
- URL: https://github.blog/changelog/2026-05-28-hard-budget-limits-now-available-for-github-advanced-security/

GitHub Advanced Security 现在支持硬预算上限，企业管理员可按 license 数设定不可超出的预算；达到阈值后，新的许可证分配会被阻止，GHAS 也不会在新仓库上继续启用。相比过去仅提供 75/90/100% 的软预算提醒，这次更新把预算从“告警”变成了“强制执行”。

Why it matters: 这不是 AI 编码技术更新，但对平台治理有现实意义：安全与 AI 工具都在走向 seat/usage 双重成本结构，组织需要可执行的预算护栏，避免自动化开通与群组同步导致失控扩张。

Key takeaways:
- 从软预算到硬预算，意味着 GitHub 开始提供更适合大组织治理的成本控制原语。
- 如果你的团队正在推广 Copilot、GHAS 或其他按席位/用量收费的开发工具，这类预算开关值得纳入统一 FinOps/PlatOps 策略。

Tags: GitHub Advanced Security, Platform Governance, FinOps, Developer Tooling
