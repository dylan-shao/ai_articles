## Improvements to secret scanning and public monitoring - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A

- URL: https://github.blog/changelog/2026-07-15-improvements-to-secret-scanning-and-public-monitoring/

GitHub shipped a set of incremental improvements to secret scanning and public monitoring: new detectors for Resend and APIclub secrets, default push protection for VolcEngine secrets, a new secret_category field in the secret_scanning_alert webhook, and insight cards in the public monitoring alert list showing leak attribution, enterprise member count, and verified domains.

Why it matters: This is not a core AI coding development, but it matters for teams running developer platforms, security governance, and automated response pipelines. The webhook classification field is especially useful for improving alert routing, prioritization, and remediation automation.

Key takeaways:
- The secret_scanning_alert webhook now includes secret_category, enabling downstream systems to distinguish default vs. generic secret types for risk-based routing.
- New public monitoring insight cards show GitHub is moving from pure detection toward governance and attribution support.
- Default push protection for more secret types reflects a broader shift from post hoc alerting to pre-commit/pre-push prevention.
