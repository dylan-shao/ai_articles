## AI credits consumed per user now in the Copilot usage metrics API
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-19
- URL: https://github.blog/changelog/2026-06-19-ai-credits-consumed-per-user-now-in-the-copilot-usage-metrics-api/

GitHub added an ai_credits_used field to the Copilot usage metrics API, exposing total AI credits consumed per user in 1-day and 28-day user-level reports at the enterprise and organization levels. The metric sits alongside existing usage reporting, making it easier to correlate spend with activity and adoption, but it is currently only an overall user total and cannot be broken down by feature, model, or surface.

Why it matters: For platform, developer productivity, and budget owners, this is a key observability step from 'is anyone using it' to 'who is consuming it and is the consumption justified by value.' It enables early cost attribution for Copilot, team-level adoption comparisons, abnormal usage monitoring, and more grounded planning for usage-based billing.

Key takeaways:
- A new ai_credits_used field is available in enterprise/org-level users-1-day and users-28-day user reports.
- The metric is derived from AI credit consumption data but is not the billing ledger and should not be treated as a billed total.
- It currently exposes only a per-user aggregate, with no breakdown by feature, model, or surface, limiting fine-grained optimization.
