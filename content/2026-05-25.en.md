## Staged publishing and new install-time controls for npm
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-22
- URL: https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm

GitHub shipped two npm supply-chain security improvements in npm CLI 11.15.0+: staged publishing is now generally available, and new install-source controls --allow-file, --allow-remote, and --allow-directory complement the existing --allow-git flag. The goal is to make package release flows more controllable and dependency source acceptance more explicit, reducing accidental publishes and risky install paths.

Why it matters: This is not an AI-coding story, but it is highly relevant to platform engineering and developer-tooling governance. As AI-generated code and automation agents increasingly add dependencies, modify build scripts, or trigger releases, install-source allowlists and progressive publishing become important guardrails against supply-chain incidents.

Key takeaways:
- npm now supports more granular allow controls over install sources, enabling teams to explicitly restrict file, remote, directory, and git-based installs.
- Staged publishing fits well into internal release-governance workflows by adding validation, observation, or approval before full public rollout.
