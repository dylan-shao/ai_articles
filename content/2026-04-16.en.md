## Enable Copilot cloud agent via custom properties - GitHub Changelog
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-15
- URL: https://github.blog/changelog/2026-04-15-enable-copilot-cloud-agent-via-custom-properties/

GitHub now lets enterprise admins selectively enable Copilot cloud agent for specific organizations instead of only using all-on, all-off, or org-decides modes. The feature is manageable through the AI Controls page or new REST API endpoints, and can use organization custom properties for initial targeting.

Why it matters: This directly affects how AI coding agents get deployed inside large enterprises. For platform and engineering governance leaders, the hard problem is rarely raw capability; it is how to pilot safely, constrain blast radius, expand gradually, and preserve policy control. This update adds a more practical adoption control plane.

Key takeaways:
- A new policy mode enables Copilot cloud agent only for selected organizations, supporting staged rollout.
- Three REST API endpoints are provided to set policy state, add organizations to the enabled list, and remove organizations from it, making platform automation easier.
- Custom properties are evaluated only once at configuration time; later property changes do not automatically recompute enablement, which matters for governance automation design.
