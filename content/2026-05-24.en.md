## Staged publishing and new install-time controls for npm
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-22
- URL: https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm

GitHub announced two supply-chain security features in npm CLI 11.15.0: staged publishing is now generally available, and new fine-grained install-source allowlist flags have been added, including --allow-file, --allow-remote, and --allow-directory, complementing the existing --allow-git flag.

Why it matters: For teams running developer platforms, build systems, and internal package ecosystems, these controls make dependency source policy enforceable. That reduces the risk of CI, agent execution environments, and automated code-change pipelines pulling unexpected local or remote content. Staged publishing also supports safer rollout and rollback for high-impact packages.

Key takeaways:
- npm now supports finer-grained install-source controls, enabling automation environments to restrict file, remote, directory, and git-based sources.
- Staged publishing helps implement progressive rollout, observation, and rollback for critical package releases, reducing supply-chain and operational risk.

## New Terms

- staged publishing: a package release mechanism that rolls out to a limited scope first, then expands progressively.
- install-source allowlist flags: command-line controls that restrict which source types dependencies may be installed from.

## Themes

- Default security boundaries in the developer toolchain are becoming more fine-grained.
- Package managers are adding policy controls better suited to enterprise governance and automated execution environments.

## Implications

- If your CI, coding agents, or internal build systems run npm install, evaluate whether unnecessary source types should be disabled by default and only enabled for allowlisted cases.
- For high-value internal packages, staged publishing should be considered as part of the release pipeline, paired with monitoring and rollback gates.
