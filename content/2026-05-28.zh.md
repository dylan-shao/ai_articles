## Copilot Memory has more controls for deletion, scope, and the Copilot CLI
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-26T21:05:28+00:00
- URL: https://github.blog/changelog/2026-05-26-copilot-memory-has-more-controls-for-deletion-scope-and-the-copilot-cli

GitHub 更新了 Copilot Memory：当用户要求遗忘某些信息时，系统会引导到正确的删除入口；新增仓库级关闭开关；并把更多记忆控制能力带到 Copilot CLI。该功能仍处于 public preview，面向所有付费 Copilot 计划开放。

Why it matters: 对工程组织而言，AI 编码助手能否长期进入真实开发流程，取决于“记住什么、在哪个范围记住、如何删除与审计”。这次更新虽然偏产品控制面，但它触及企业采用的核心阻力：隐私、仓库边界、团队治理和 CLI 工作流的一致性。

Key takeaways:
- “记忆”正在从单纯的个性化能力，转向需要明确边界、删除语义和仓库级治理的企业功能。
- CLI 纳入记忆控制意味着 AI 助手不再只服务于 IDE 内交互，而是在终端与自动化脚本链路中也需要一致的治理模型。
