## New features and Claude as agent provider preview in JetBrains IDEs
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-06-22
- URL: https://github.blog/changelog/2026-06-22-new-features-and-claude-as-agent-provider-preview-in-jetbrains-ides/

GitHub 在 JetBrains IDE 中更新了 Copilot agent 能力：支持直接使用组织级和企业级自定义 agents，Copilot CLI 会话可排队与 steering，新增 agent debug logs 摘要视图，并将 Claude 作为 agent provider 引入公开预览；同时 cloud agent 进入 GA，并增加每轮 AI credits 指示与 model picker 改进。

Why it matters: 这条更新的核心价值在于“企业可治理的 agent 分发与运行面板”开始成型。对工程平台团队来说，组织管理员可发布标准化 agents，开发者在 IDE 内直接消费，意味着 prompt/工具链/工作流可以从个人配置转为团队资产；而 CLI steering、debug logs 摘要和 credits 可见性，则补上了人机协作、故障排查与成本治理三个关键环节。

Key takeaways:
- 组织级/企业级 agents 直接进入 JetBrains，说明 agent 正在被当作可集中发布和治理的开发平台构件，而不只是个人聊天配置。
- CLI steering、debug logs 摘要、per-turn credits 指示一起出现，表明 agent 产品正在补足可控性、可观测性和 FinOps 能力。

## New Terms

- 组织级/企业级 agents：由 GitHub 组织或企业管理员统一发布、供成员直接使用的自定义代理。
- CLI steering：在命令行 agent 会话执行过程中，对消息流和执行方向进行排队、引导或调整的控制能力。
- agent debug logs 摘要视图：将代理运行日志压缩成可读摘要，用于快速诊断工具调用与执行问题。
- per-turn AI credits indicator：在每轮交互级别展示 AI 配额/信用消耗的可见性机制。

## Implications

- 平台团队应开始评估是否把内部编码规范、常见修复流程和 repo 操作封装成可集中发布的 agents，而不是依赖个人 prompt 习惯。
- 若团队已试点 CLI agent，需要同步建设日志留存、运行审计和成本归因，否则规模化后会出现调试与治理盲区。
- 采购或标准化 IDE/agent 工具时，应把多模型接入放在次要位置，把 agent 分发、权限边界、日志与成本指标放在优先位置。
