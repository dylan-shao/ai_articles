## GitHub Copilot in Visual Studio Code, May releases
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-06-03
- URL: https://github.blog/changelog/2026-06-03-github-copilot-in-visual-studio-code-may-releases/

这篇月度更新最有价值的部分不是零散新功能，而是把 VS Code/Copilot 的 agent 执行面逐渐产品化：Agents window 进入 Stable 预览，支持跨项目任务导向操作；远程 agent 可通过 SSH 或 Dev Tunnels 在远端机器上持续运行；BYOK 扩展到 air-gapped 环境，并允许为不同辅助任务指定模型；同时继续推进 Agent Host Protocol，用于跨客户端同步 agent 会话状态。

Why it matters: 这说明 IDE 内的 AI coding 正在从“本地交互式助手”演进为“可托管、可远程、可同步、可治理”的 agent 系统。对平台团队来说，AHP、远程会话和 BYOK/隔离能力比单个模型升级更关键，因为它们直接决定企业是否能把 agent 接入内网、受控执行环境和现有开发治理流程。

Key takeaways:
- Agents window 把任务执行、跨项目导航和变更审查整合成 agent-first 工作台，意味着 IDE 正在重新围绕代理工作流设计。
- 远程 agent 与断线后继续运行的能力，指向长期任务、后台执行和更接近 CI/开发容器的执行模型。
- BYOK 进入 air-gapped 环境并支持按任务路由模型，反映企业部署对合规、成本与模型控制的要求正在成为核心能力。
- Agent Host Protocol 是最值得跟踪的底层信号之一：它可能成为多客户端、多宿主之间共享 agent 会话状态的关键接口层。

## New Terms

- Agent Host Protocol (AHP)：用于在多个客户端之间同步代理会话状态的开放协议方向。
- BYOK in air-gapped environments：在隔离/内网环境中使用自带模型密钥的企业部署模式。
- Remote agents：通过 SSH 或 Dev Tunnels 在远端机器持续运行的编码代理会话。

## Themes

- AI coding 从聊天式补全转向任务式、长会话、远程执行的 agent runtime。
- 企业采用的关键约束正从模型能力转向协议、隔离、会话持久化和治理控制。
- IDE、远程宿主与 Git 工作流正在被整合为统一的代理执行面。

## Implications

- 如果你在建设内部编码代理平台，应优先评估会话状态模型、远程执行宿主、断线恢复以及与 Git/代码评审的耦合点，而不是只比较模型排行榜。
- 需要尽早定义模型路由与权限边界：哪些任务可走外部模型，哪些必须在内网/沙箱执行。
- 值得持续关注 AHP 或类似协议是否开放成熟，因为这会影响多 IDE、CLI、Web 端统一代理体验的可行性。
