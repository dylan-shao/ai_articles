## New features and Claude as agent provider preview in JetBrains IDEs
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-06-22
- URL: https://github.blog/changelog/2026-06-22-new-features-and-claude-as-agent-provider-preview-in-jetbrains-ides/

GitHub updated Copilot agent capabilities in JetBrains IDEs: org- and enterprise-level custom agents are now usable directly in the IDE, Copilot CLI sessions support message queueing and steering, a new agent debug-log summary view was added, Claude is now in public preview as an agent provider, and cloud agent is generally available alongside per-turn AI credit indicators and model-picker improvements.

Why it matters: The main value is that a governable enterprise agent distribution and runtime surface is taking shape. For platform teams, admins can publish standardized agents that developers consume directly in the IDE, turning prompts/toolchains/workflows from personal setup into team assets. CLI steering, debug-log summaries, and credit visibility add the missing layers for human-in-the-loop control, troubleshooting, and cost governance.

Key takeaways:
- Org/enterprise agents landing directly in JetBrains shows agents are becoming centrally published and governed platform artifacts, not just personal chat presets.
- CLI steering, debug-log summaries, and per-turn credit indicators together signal that agent products are maturing on controllability, observability, and FinOps.

## New Terms

- Org/enterprise agents: custom agents published centrally by GitHub organization or enterprise admins for members to use directly.
- CLI steering: the ability to queue, guide, or adjust message flow and execution direction during command-line agent sessions.
- Agent debug-log summary view: a compressed, readable summary of agent runtime logs for faster diagnosis of tool calls and execution issues.
- Per-turn AI credits indicator: visibility into AI credit consumption at the level of each interaction turn.
