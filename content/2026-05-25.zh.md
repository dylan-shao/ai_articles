## Staged publishing and new install-time controls for npm
- Source: GitHub Changelog
- Author: Allison
- Publish date: 2026-05-22
- URL: https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm

GitHub 在 npm CLI 11.15.0+ 中推出两项供应链安全改进：分阶段发布（staged publishing）正式可用，以及新的安装来源控制参数 --allow-file、--allow-remote、--allow-directory，与既有的 --allow-git 配合使用。核心目标是让包发布更可控、让依赖安装来源更显式，从而减少误发布与高风险来源被默认接受的情况。

Why it matters: 虽然这不是 AI coding 主题，但它与平台工程和开发者工具治理高度相关。随着 AI 生成代码和自动化代理越来越频繁地引入依赖、修改构建脚本或触发发布流程，安装来源白名单与渐进式发布机制会成为防止供应链事故的重要护栏。

Key takeaways:
- npm 现在支持更细粒度的安装来源许可控制，团队可以显式限制 file、remote、directory、git 等来源。
- 分阶段发布适合接入内部发布治理流程，在全量公开前增加验证、观察或审批环节。
