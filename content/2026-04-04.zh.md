## Organization runner controls for Copilot cloud agent
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-03
- URL: https://github.blog/changelog/2026-04-03-organization-runner-controls-for-copilot-cloud-agent/

GitHub 为 Copilot cloud agent 增加了组织级 runner 控制。此前 agent 的执行环境主要在仓库级通过 copilot-setup-steps.yml 配置；现在组织管理员可以为全组织设置默认 runner，并可锁定该设置，禁止仓库覆盖。agent 仍然基于 GitHub Actions 启动临时开发环境，但现在可以统一指定标准托管 runner、大型 runner 或 self-hosted runner。

Why it matters: 这是 agent 进入企业平台治理的关键一步。runner 决定了性能、成本、网络可达性、凭据边界与内部资源访问方式。把配置从 repo 级提升到 org 级，意味着平台团队终于可以像管理 CI 执行面一样管理 AI agent 的执行面，降低碎片化配置和安全漂移。

Key takeaways:
- Copilot cloud agent 的执行环境现在可由组织统一设定默认 runner，而不再依赖每个仓库单独配置。
- 组织可以锁定 runner 策略，确保 agent 始终运行在指定的 GitHub-hosted、large runner 或 self-hosted 环境中。

## Organization firewall settings for Copilot cloud agent
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-03
- URL: https://github.blog/changelog/2026-04-03-organization-firewall-settings-for-copilot-cloud-agent/

GitHub 为 Copilot cloud agent 增加了组织级防火墙设置。该 agent 内置 firewall，用于限制互联网访问并降低 prompt injection 与数据外泄风险。现在组织管理员可以统一控制全组织范围内是否启用防火墙、是否启用推荐 allowlist、维护组织级自定义 allowlist，以及决定仓库管理员能否额外添加 allowlist 条目。

Why it matters: 这直接回应了 agent 落地中的核心安全顾虑：一旦 agent 能自主浏览、检索、拉依赖、访问外部资源，就必须有明确的网络边界与默认 deny/allow 策略。把这些能力上收到组织层，说明 GitHub 正把 AI coding agent 视为需要零信任治理的执行体，而非普通 IDE 助手。

Key takeaways:
- 组织管理员可统一控制 agent 是否能联网、用什么推荐 allowlist，以及是否允许仓库自行扩展规则。
- 该功能的目标是缓解 prompt injection 和数据外泄风险，并让 agent 在大规模组织内可控上线。

Tags: Copilot cloud agent, agent firewall, prompt injection, security governance
Scores: signal 9/5, novelty 8/5, actionability 9/5

## Copilot cloud agent signs its commits
- Source: GitHub Changelog
- Author: N/A
- Publish date: 2026-04-03
- URL: https://github.blog/changelog/2026-04-03-copilot-cloud-agent-signs-its-commits/

Copilot cloud agent 现在会对其生成的每个 commit 进行签名，并在 GitHub 上显示为 Verified。这样一来，启用了“Require signed commits”分支保护规则的仓库也可以使用该 agent；此前这类规则会直接阻止 agent 提交。

Why it matters: 这是一个小更新，但对企业落地极其关键。很多受监管或高安全团队默认启用签名提交策略；如果 agent 不满足这一基础门槛，就无法进入主流受控仓库。提交签名使 agent 生成的变更首次真正融入现有供应链可信与审计链条。

Key takeaways:
- agent 生成的提交现在具备可验证身份，能与 GitHub 的 Verified 提交机制对接。
- 启用 signed-commit 分支保护的仓库不再天然排斥 Copilot cloud agent。

Tags: Copilot cloud agent, signed commits, branch protection, supply chain trust
