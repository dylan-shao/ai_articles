### [警告] 1. Self-hosted Cloud Agents
- Title: Self-hosted Cloud Agents
- Source: Cursor Changelog
- Author: 未标注
- Publish date: 2026-03-25
- URL: [cursor.com/changelog](https://cursor.com/changelog)

**Summary**  
Cursor 发布了 self-hosted cloud agents，强调代码、构建产物和密钥都可保留在企业自有网络内，而 agent 仍能在本地基础设施中执行工具调用。它本质上是在回答一个企业级 adoption 难题：如何把 cloud-agent 体验带进受控网络，而不是把仓库和执行环境外包出去。

**Why it matters**  
这对平台团队有现实意义，因为“是否能在自有网络内运行 agent”往往比模型能力本身更决定企业落地速度。但它以产品能力公告为主，技术细节深度有限，而且发布时间已超出今天的 24 小时窗口。

**Key takeaways**
- 自托管执行面正在成为企业级 coding agent 的关键部署形态。
- 隔离 VM、完整开发环境、插件和多模型 harness 正在从“平台特性”变成“准入门槛”。
- 安全边界、网络边界和工具执行边界，已经成为 agent 产品差异化的核心。

**Applicable tags**

- `self-hosting`
- `enterprise-adoption`
- `execution-environment`

**Summary**  
OpenAI 在企业工作区里提供了 workspace analytics，用于观察用户活跃度、功能采用情况和整体使用趋势。它不是 coding-specific 更新，但对想要规模化推动 AI coding 的组织来说，补上了 adoption 可观测性这一层。

**Why it matters**  
很多团队的问题不是“有没有 agent”，而是“哪些团队真的在用、用了哪些能力、是否形成稳定工作流”。不过这条更新更偏运营与治理，而非 coding workflow、benchmark 或 orchestration 的新增技术内容。

**Key takeaways**
- AI coding 规模化落地需要产品使用分析，而不只是模型效果评估。
- 平台团队需要把活跃度、功能采用率和团队分布纳入内部运营仪表盘。
- 可观测性层对 seat 扩张、培训投入和 ROI 复盘有直接价值。

- `self-hosted cloud agents`
  - 在企业自有网络中保留代码、构建产物和密钥，同时维持 cloud-agent 体验的部署模式。
- `workspace analytics`
  - 面向 AI 工具落地的组织级采用度与使用行为观测层。
- `execution boundary`
  - 对 coding agent 来说，执行发生在哪里、能访问哪些工具和网络，正变成与模型能力同等重要的系统设计问题。
