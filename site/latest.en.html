Quantifying infrastructure noise in agentic coding evals

  • Source: Anthropic Engineering
  • Author: N/A
  • Publish date: N/A
  • URL: https://www.anthropic.com/engineering/infrastructure-noise

Anthropic examines “infrastructure noise” in agentic coding benchmarks. Unlike static benchmarks, tasks such as SWE-bench and Terminal-Bench require models to operate inside a live runtime—writing code, running tests, installing dependencies, and iterating across turns—so the environment itself changes outcomes. The post reports that, in internal Terminal-Bench 2.0 experiments, the gap between the best- and worst-resourced setups reached about 6 percentage points, larger than many leaderboard margins.

Why it matters: This reframes coding evals from pure model comparison to systems design. If teams run the same benchmark with different CPU/RAM limits, timeout policies, container isolation, or enforcement methods, they are not actually administering the same test. For organizations building internal AI coding platforms, choosing vendor models, or defining engineering productivity metrics, infra configuration must be treated as a versioned, auditable part of the benchmark itself.

Key takeaways: - Agentic coding eval outcomes are not determined by the model alone; they depend materially on runtime resource budgets and execution policy. - Infra-induced score swings of several percentage points are large enough to overturn leaderboard interpretations, so benchmark reproduction requires explicit and enforced resource specs. - Evaluation harnesses should log CPU, memory, timeout, concurrency, scheduling, and failure modes, or conclusions may not be comparable.
